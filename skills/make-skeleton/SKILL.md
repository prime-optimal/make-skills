# Make Skeleton

Create a valid starter blueprint plan from minimal requirements.

## Inputs

- Trigger type
- Expected outputs
- Rough behavior description

## Outputs

- Starter module layout
- Module ordering and routing plan
- Initial mapper/interface scaffold notes

## Rules

- Follow Vinny naming and pattern conventions.
- Include Data Store usage where conversational state is required.
- Include approval-flow integration when messaging/draft send paths are involved.
